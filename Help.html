MarkNote


  • Comprehensive and truly flexible multi-format export, you can export your notes into HTML or PDF format, you can even use it to build static web sites
  • Live preview while writing
  • Syntax highlighting for Markdown and code
  • Safe and convenient document management
  • Your notes synced into iCloud drive, and share among all your devices, mac, iPhone and iPad
  • Organize your documents with tags
  • Instant search on your notes

Advanced usage

Create Table of Contents (TOC)

To create a Table of Contents (TOC), just create a new line and then type [TOC].The “Table of Contents” will extract all headers from your writing, its contents will be updated automatically.

Refer to another note

You can refer to another note with the syntax of:

[[{title}]]

Where {title} is the title of the note you want to refer to.

Use tags to organize your notes

Tags are a good way to keep your notes organized.
To add a tag to your note, just add a line under the title of your note, in the format of
Tags:{tag1},{tag2}One note can be associated with multiple tags.

Math typesetting

MarkNote supports rendering mathematics in both inline mode and block mode. The rendering is processed by MathJax.
To create an inline formula, just wrapper your formula with $. For example, $ E = mc^2 $ should be rendered as E = mc^2

Similarly, to create a formula block, just wrapper your formula with $$.MarkNote supports 3 types of Math notation: AsciiMath, Tex/LaTeX and MathML.

AsciiMath

Check here for AsciiMath syntax.
Sample:
Code:

$$
sum_(i=1)^n i^3=((n(n+1))/2)^2
$$

Will be rendered as:

$$ sum_(i=1)^n i^3=((n(n+1))/2)^2 $$

TeX, LaTeX

You could find the all supported TeX/LaTeX commands here
Sample:
Code:

$$
\displaystyle x = \frac{ - b \pm \displaystyle\sqrt {b^2 - 4ac} }{2a}
\ {\text{ when}}\ {ax^2 + bx + c = 0}
$$

will be rendered as:

$$ \displaystyle x = \frac{ - b \pm \displaystyle\sqrt {b^2 - 4ac} }{2a} \ {\text{ when}}\ {ax^2 + bx + c = 0} $$

Use Marknotes to build static blog website ( iOS version support is in progress )

Do you believe my blog site is completed done with mac version MarkNotes?
You can also build your own with just several clicks!

From "File" menu, find "Export All Notes as Blog Site", click it, an Export wizard will show, with 3 tabs below:

  • "1:Rendering", for you to specify the output style, light or dark etc;
  • "2:Site Settings", for you to input some basic information about your site, and also to filter which notes you want to export/hide;
  • "3:Export Destination", for you to specify where is the output folder;

In the "2:Site Settings", there are a few options you need to pay attention:

Item Description
SiteName The name of your site
Base URL The root URL of your site
SiteDesc The description of your site, will show at the bottom of each page
github-user your github account, optional
twitter-user Your account name on Twitter, optional
hidden-tagsTags you don't want to export when generating static site, use comma (,) to separate multiple values
export-tagsIf this value exists, only notes with these tags will be exported when generating static site, use comma (,) to separate multiple values

Once configured, just click the "Export" button in "Export Destination" tab, and you will get a folder which can be put to any HTTP server.

FAQ

  1. Why the static site generated can be browsed, but links to other pages do not work?
Answer: The navigation links need some javascript, which cannot work in local mode. You will need a Web server to run the static site.

  1. How to change the font size?
Answer: MarkNote for iOS will adjust font size based on iOS preference. To adjust your font size, open "Settings", then "General", then "Accessibility", and click "Larger Text".
So far, MarkNotes for mac cannot change font size yet.

  1. Where are my notes stored?
Answer:
First of all, for iOS and mac editions, all your notes are stored locally in your device's file system.
On macOS, the location is: /Users/{You user name}/Library/Containers/marknote.MarkNoteMac/Data/Documents

If iCloud is enabled, a copy of all your data will also be stored into iCloud. Your notes will be stored into iCloud private database. Only you have access to your data. Even the developer is not be able to see your notes.

Your published notes/blog posts will be stored in Apple's CloudKit public database.
